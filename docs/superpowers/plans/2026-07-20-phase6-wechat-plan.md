# Phase 6 — 微信小程序实现计划

> 日期：2026-07-20
> 设计文档：`docs/superpowers/specs/2026-07-20-phase6-wechat-design.md`
> 前置条件：Phase 1-5 已完成

## 任务清单

### 任务 1：暗色模式基础设施 + CSS 变量迁移

**目标：** 建立 CSS 变量体系，所有颜色从硬编码迁移到变量，支持暗色模式切换。

**步骤：**
1. 创建 `utils/theme.ts` — 主题工具（getTheme/setTheme/applyTheme）
2. 修改 `app.wxss` — 定义 CSS 变量（亮色 + 暗色），所有全局样式用变量
3. 修改 4 个现有页面的 `.wxss` — 硬编码颜色替换为 `var(--xxx)`
4. 修改 `app.ts` — `onLaunch` 中应用主题
5. 每个页面的 `onShow` 中调用 `applyTheme` 同步导航栏

**提交：** `feat(wx): 暗色模式基础设施 + CSS 变量迁移（任务 1/6）`

---

### 任务 2：API 类型补全 + 角色管理 API

**目标：** 在 `api.ts` 中补全 Role/Menu/Dictionary 类型定义和 API 方法。

**步骤：**
1. 在 `api.ts` 添加 `RoleItem`、`MenuItem`、`DictItem` 接口
2. 添加角色 CRUD API 方法
3. 添加菜单列表 API 方法
4. 添加字典列表 API 方法
5. 添加用户启用/禁用 API 方法

**提交：** `feat(wx): API 类型补全 + 角色/菜单/字典 API 封装（任务 2/6）`

---

### 任务 3：角色管理页面

**目标：** 新增角色管理页面，支持列表/搜索/新增/编辑/删除。

**步骤：**
1. 创建 `pages/roles/` 四件套（.ts/.wxml/.wxss/.json）
2. 实现列表 + 分页 + 搜索
3. 实现新增/编辑弹窗
4. 实现删除确认
5. 在 `app.json` 注册页面和 tabBar
6. 页面 `onShow` 应用主题

**提交：** `feat(wx): 角色管理页面（列表/搜索/CRUD）（任务 3/6）`

---

### 任务 4：菜单管理页面（只读树）

**目标：** 新增菜单管理页面，树形展示菜单结构。

**步骤：**
1. 创建 `pages/menus/` 四件套
2. 调用 `/api/menu/list` 获取平铺列表
3. 前端构建树结构（递归 + parent_id）
4. 使用缩进 + 折叠图标展示层级
5. 点击菜单项展示详情
6. 在 `app.json` 注册页面
7. 在 profile 页面添加入口

**提交：** `feat(wx): 菜单管理页面（树形只读）（任务 4/6）`

---

### 任务 5：字典管理页面 + 用户启用/禁用

**目标：** 新增字典管理页面，用户列表添加启用/禁用操作。

**步骤：**
1. 创建 `pages/dicts/` 四件套
2. 实现字典列表 + 搜索
3. 点击字典项查看详情
4. 修改 `users` 页面 — 添加用户启用/禁用操作（点击 → 弹出 ActionSheet → 调用 PUT）
5. 在 `app.json` 注册页面
6. 在 profile 页面添加入口

**提交：** `feat(wx): 字典管理页面 + 用户启用/禁用操作（任务 5/6）`

---

### 任务 6：UI 打磨 + 暗色模式入口 + 最终验证

**目标：** 加载骨架屏、空状态优化、profile 页暗色切换开关、最终验证。

**步骤：**
1. 在 dashboard 和 users 页面添加骨架屏
2. 统一空状态样式
3. 添加 `hover-class` 交互反馈
4. 在 profile 页面添加暗色模式切换开关
5. 验证所有页面暗色模式显示正常
6. 后端 `cargo clippy` + `cargo test` 回归

**提交：** `feat(wx): UI 打磨 + 暗色模式切换 + 最终验证（任务 6/6）`

---

## 执行顺序

```
任务 1 (暗色基础) → 任务 2 (API 类型) → 任务 3 (角色管理)
                                        → 任务 4 (菜单管理)
                                        → 任务 5 (字典+用户操作)
                                        → 任务 6 (UI打磨+验证)
```

任务 1 和 2 是基础，必须先完成。任务 3-5 依赖任务 2 的 API 定义。任务 6 最后收尾。
